# SEO Implementation Guide for Next.js Portfolio

## Overview

This guide explains the production-grade SEO implementation in your Next.js 16+ portfolio. All files have been set up to provide comprehensive SEO optimization including metadata, Open Graph tags, structured data (JSON-LD), and technical SEO best practices.

## 2. Root Layout (`src/app/layout.tsx`)

### Purpose
Implements base metadata and global structured data that applies to all pages.

### What It Does
- ✅ Exports base metadata using `constructMetadata()`
- ✅ Includes preconnect hints for performance
- ✅ Injects website schema JSON-LD in `<head>`
- ✅ Uses optimized font loading with `display: 'swap'`
- ✅ Wraps app in ThemeProvider

### Key Decisions
- **Font optimization**: `display: 'swap'` prevents Cumulative Layout Shift (CLS)
- **Preconnect**: Links to Google Fonts for better performance
- **JSON-LD**: Website schema helps search engines understand your site structure

## 3. Sitemap (`src/app/sitemap.ts`)

### Purpose
Generate dynamic XML sitemap for search engines to discover all pages.

### Automatic Generation
- ✅ Accessible at `/sitemap.xml` (auto-generated by Next.js)
- ✅ Includes all static routes with priorities
- ✅ Can be extended with dynamic routes

### Update with Your Routes
```typescript
// Add your actual project routes here
const projectRoutes: MetadataRoute.Sitemap = [
  {
    url: `${baseUrl}/projects/my-project`,
    lastModified: new Date(),
    changeFrequency: 'monthly',
    priority: 0.8,
  },
]
```

### Priority Levels
- `1.0` → Homepage (most important)
- `0.9` → Projects section
- `0.8` → Individual projects or main sections
- `0.7` → Supplementary pages

## Performance Optimization Tips

### Image Optimization
- ✅ Use Next.js `<Image>` component (automatic optimization)
- ✅ Include descriptive `alt` text (SEO + accessibility)
- ✅ Set `priority={true}` for above-the-fold images
- ✅ Use proper aspect ratios with `width` & `height`

### Font Optimization
- ✅ Already implemented with `display: 'swap'`
- ✅ Limits subsets to needed languages (Latin + Bengali)

### Code Splitting
Use dynamic imports for heavy components:
```typescript
import dynamic from 'next/dynamic'

const HeavyComponent = dynamic(() =>
  import('@/components/HeavyComponent'),
  { loading: () => <div>Loading...</div> }
)
```

### Core Web Vitals
- **LCP** (Largest Contentful Paint): Preload critical images
- **FID** (First Input Delay): Code-split heavy JS
- **CLS** (Cumulative Layout Shift): Use font `display: swap`, define image dimensions

## Common Pitfalls & Solutions

| Issue | Solution |
|-------|----------|
| Multiple H1 tags | Use only one `<h1>` per page |
| Missing alt text | Always add descriptive `alt` to images |
| Broken canonical URLs | `constructMetadata()` handles this automatically |
| No structured data | Use `<JsonLd />` component or root layout injection |
| Image layout shift | Define `width` and `height` props |
| Slow OG image | Already optimized with Edge Runtime |
| Missing sitemap | Auto-generated at `/sitemap.xml` |

## Advanced: Custom Project Schemas

To add custom structured data for specific project types:

```typescript
// projects/[slug]/page.tsx
function generateCustomSchema(project) {
  return {
    '@context': 'https://schema.org',
    '@type': 'SoftwareApplication',  // Or 'CreativeWork', 'Product', etc.
    name: project.name,
    description: project.description,
    author: { '@type': 'Person', name: 'Your Name' },
    applicationCategory: 'WebApplication',
    url: project.liveUrl,
    sourceCode: project.githubUrl,
    programmingLanguage: ['JavaScript', 'TypeScript', 'React'],
    // Add more properties as needed
  }
}
```
